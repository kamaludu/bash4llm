BASH4LLM(1)                      Bash4LLM Manual                    BASH4LLM(1)

NAME
    bash4llm - Resilient and secure Bash-first wrapper for querying external 
    LLM providers (v2.6.0)

SYNOPSIS
    bash4llm [OPTIONS] [PROMPT]
    bash4llm -f FILE [OPTIONS]
    echo "prompt" | bash4llm [OPTIONS]
    bash4llm --batch BATCH_FILE [OPTIONS]
    bash4llm --chat [OPTIONS]
    bash4llm --vault

DESCRIPTION
    bash4llm is a native shell-environment command-line utility designed for 
    flexible integration, automation, and orchestration of interactions with 
    large language models (LLMs) provided by external services such as Groq, 
    Gemini, and HuggingFace.

    The runtime operates under restrictive execution constraints (set -euo 
    pipefail) to ensure the immediate termination of pipelines when encountering 
    invalid commands or uninitialized variables. The architecture is entirely 
    designed to isolate system operations from remote execution, prohibiting 
    any dynamic interpretation or local execution of generated code (zero eval 
    policy) and excluding the use of shared system directories like /tmp to 
    mitigate symlink attacks (symlink attack mitigation).

OPTIONS
    Script options are grouped by functional category. Parameters can be passed 
    in any arbitrary order, except for the end-of-options marker.

  Section 1: Input and Prompt Management
    -f FILE
        Adds the specified file as a context source for processing. Can be 
        specified multiple times to concatenate multiple input files 
        sequentially. Individual files are separated by structured visual 
        demarcators to allow the model to clearly distinguish sources.

    --json-input FILE
        Sends a structured JSON payload, useful for directly defining a list of 
        custom messages with specific roles. If set to '-', the script reads 
        from standard input (stdin). Using this option excludes and clears the 
        generic prompt variable resolution.

    --template NAME
        Indicates the text template to apply to the user prompt, sourced from 
        the user template directory. The template file must contain the 
        placeholder "{{CONTENT}}" which is replaced at runtime with the 
        assembled user input text.

    --batch FILE
        Activates batch processing mode. The script parses the specified file 
        and executes a distinct API call for each non-empty line, excluding 
        comments (lines starting with a hash symbol). An error in a single 
        prompt does not halt the overall batch sequence.

    PROMPT (Positional Arguments)
        All positional arguments not captured by specific flags are treated as 
        text fragments. If an argument corresponds to a regular, readable file 
        path on disk, the script expands it by reading and incorporating its 
        content; otherwise, it interprets it as literal text to append to the 
        main prompt.

  Section 2: Model and Generation Parameters
    -m, --model MODEL
        Specifies the name of the LLM model to query. This setting takes 
        absolute precedence over locally saved default models.

    --system TEXT
        Defines or overrides the system prompt, which instructs or conditions 
        the behavior, tone, and operational context of the language model.

    --temperature VALUE, --ture VALUE
        Sets the decimal sampling temperature (usually between 0.0 and 2.0). 
        Both flags link and synchronize bidirectionally to the internal 
        variables "TURE" and "TEMPERATURE", unifying their internal resolution 
        before transmission.

    --max NUMBER
        Sets the maximum limit of tokens to generate in the model's response 
        (internal variable "MAX_TOKENS").

  Section 3: Thread and Conversation Management
    --thread THREAD_ID
        Enables multi-turn chat context persistence by associating the history 
        with a unique identifier. The ID is formally validated and 
        cryptographically anonymized via SHA-256 ($SAFE_THREAD_ID) to prevent 
        PII leakage on disk. The ID must match the regex ^[A-Za-z0-9._-]{1,128}$.

    --thread-window [WINDOW]
        Specifies the maximum number of previous historical messages to retrieve 
        from the thread history file to compose the context window. If the 
        flag is specified without arguments, the default value is set to 10. 
        To optimize performance and avoid saturating the model's context, 
        values less than or equal to 20 are recommended; setting values greater 
        than 20 causes the runtime to emit a diagnostic warning.

    --init-thread (Requires --thread <id>)
        Allows external components (such as a CGI graphical interface) or 
        automation scripts to pre-register a new thread in the system before 
        sending the first prompt. This option performs local thread registration 
        and terminates execution immediately (exit 0) without making API calls.

    --delete-thread THREAD_ID
        Permanently wipes active thread history files (.ndjson), lock files, 
        metadata files (.json), and removes the thread ID entry from the global 
        thread index under an exclusive lock.

    --rename-thread THREAD_ID
        Updates the user-facing title of the specified thread inside its 
        metadata file. Must be used in conjunction with --title <text>.

    --title TEXT
        Specifies the new user-facing title string when executing thread 
        renaming via --rename-thread.

  Section 4: Output and Formatting
    --json
        Disables text extraction and displays the raw JSON response payload 
        from the provider exactly as received from the external API.

    --no-color
        Disables the use of ANSI colors and text styles on both standard output 
        and standard error, ensuring clean logs in non-interactive sessions.

    --pretty
        Performs structured formatting (pretty-print) on the JSON response 
        before displaying it on standard output.

    --text
        Selects standard text display mode (default). The script extracts only 
        the textual message body produced by the model.

    --raw
        Returns only the response text, excluding any additional trailing 
        newline characters.

    --out PATH
        Saves the final response to the specified path. If the path points to 
        an existing directory, the script generates a unique file named with 
        a chronological timestamp inside it; if it points to a file, it 
        overwrites it transactionally and securely.

    --threshold NUMBER
        Sets the threshold limit, expressed in text characters (default 1000 
        characters), above which the response produced by the LLM is 
        automatically and permanently saved in the interaction history directory.

    --save
        Forces unconditional saving of the textual result to the history 
        directory, ignoring the threshold limit check (FORCE_SAVE_MODE="save").

    --nosave
        Absolutely disables automatic archiving of the result in the 
        interaction history (FORCE_SAVE_MODE="nosave").

  Section 5: Connection Flows and Network
    --stream
        Enables real-time token reception (data streaming). Requires the active 
        provider to support the SSE (Server-Sent Events) connection interface. 
        This option is mutually exclusive with structured --json and --pretty 
        formats.

    --no-stream
        Explicitly inhibits and disables token streaming mode, restoring 
        synchronous, blocking reception of the entire response.

    --dry-run
        Simulates the transaction without making real network connections. The 
        script compiles the final JSON request payload structure, displays its 
        first few lines on standard error for diagnostic purposes, and 
        terminates execution with a success status.

  Section 6: Module Selection, Security and Installation
    --provider [NAME], --providers [NAME]
        If specified with a name, selects and sets the active provider, 
        persistently saving the preference in the configuration directory. If 
        the passed value is "list", or if the flag is specified without any 
        argument in an interactive TTY, opens an interactive selection menu.

    --vault
        Launches the interactive cryptographic console tool to manage encrypted 
        API keys inside keys.enc, supporting key creation, editing, deletion, 
        master password changes, and emergency offline recovery.

    --install-extras [SOURCE], --install-extra [SOURCE]
        Installs and extends the runtime by copying modules and resources 
        from the extensions directory into the persistent user extensions folder, 
        performing strict SHA-256 manifest integrity checks.

  Section 7: Model Management and Quick Configuration
    --refresh-models, --refresh-model
        Forces an immediate connection with the active provider's API to 
        synchronize the local list of available models, saving it atomically 
        and securely under an exclusive lock in the models file.

    --list-models, --list-model
        Displays a formatted, readable list of text models currently available 
        and ready for use in the local system.

    --list-models-raw
        Returns the raw list of local models in machine format (one normalized 
        identifier per line) for automation purposes.

    --list-providers, --list-provider
        Displays the list of provider modules installed in the system.

    --list-providers-raw
        Returns the list of installed providers in machine format, without 
        formatting or text headers.

    --set-default MODEL
        Sets and permanently archives the specified model as the default choice 
        for the currently active provider in the system.

  Section 8: Interactive UI, Diagnostics and Inspection
    --chat, --tui
        Launches the full-screen TUI interactive REPL session for continuous 
        conversations.

    --quiet
        Runtime silencer. Inhibits all informational prints and non-critical 
        warnings on standard error.

    --debug
        Enables advanced diagnostics mode for developers. Writes detailed log 
        messages to standard error and preserves temporary runtime files.

    --show-config
        Generates a structured tabular view of current runtime parameters 
        (software versions, selected provider, active model, operational paths, 
        and synchronization file states).

    --check-config
        Executes a static security audit on the user configuration file, 
        checking file system permissions for group/world write vulnerabilities 
        and linting variable keys against canonical specifications.

    --diagnostics, --diagnostic
        Starts a complete self-check inspection cycle verifying physical folders, 
        write access, provider module integrity, API keys, and testing TLS 
        connection handshakes via OpenSSL.

    --explain-error CODE_OR_ALIAS
        Explains numerical error exit codes (e.g. 15, 17) or string aliases 
        (e.g. BASH4LLM_ERR_SEC) using documented definitions and operational 
        mitigation tips.

    --print-config-dir
        Prints the canonical configuration directory path to standard output.

    --print-provider-file
        Prints the canonical active provider persistence file path to standard output.

    --print-model-file PROVIDER
        Prints the canonical default model file path for the specified provider 
        to standard output.

    --version
        Prints script identifier name, software version, release date, and exits.

    -h, --help
        Displays the quick start interface guide and terminates.

    --
        End-of-options marker. Any argument positioned after this delimiter is 
        interpreted as input prompt text.

ENVIRONMENT VARIABLES
    The behavior of the bash4llm runtime is governed and can be conditioned by 
    the following environment variables.

  Section 1: API Credentials and Vault Security
    GROQ_API_KEY
        Default API key for authenticating network calls to the built-in Groq 
        provider.

    PROVIDER_API_ENV_[PROVIDER]
        Indirect mapping for external providers. Defines the name of an 
        alternative environment variable containing the API key for a given 
        custom provider.

    BASH4LLM_VAULT_ENABLED
        Boolean flag (1 or 0) to control the cryptographic key vault system. 
        Set to 0 to permanently or temporarily disable vault lookups (default: 1).

    _B4L_RT_CTX
        Obfuscated session context token held in active shell memory to unlock 
        the Master Vault Key and bypass password prompts across multiple runs.

    BASH4LLM_OPENSSL_ACTIVE
        Internal state flag (1 or 0) indicating whether the OpenSSL security 
        helper extension is loaded and active.

  Section 2: Paths and Isolated File System Directories
    BASH4LLM_DIR
        Persistent root directory for bash4llm resources and data. Defaults to 
        "bash4llm.d" relative to the script location.

    BASH4LLM_ROOT
        Alternative path used to derive the root directory if BASH4LLM_DIR is 
        not explicitly declared.

    BASH4LLM_CONFIG_DIR
        Directory storing persistent user settings (default: $BASH4LLM_DIR/config).

    BASH4LLM_MODELS_DIR
        Directory containing configured and cached models (default: $BASH4LLM_DIR/models).

    BASH4LLM_TEMPLATES_DIR
        Directory containing user template files (default: $BASH4LLM_DIR/templates).

    BASH4LLM_HISTORY_DIR
        Directory containing response history and thread logs (default: $BASH4LLM_DIR/history).

    BASH4LLM_TMPDIR
        Protected root directory for temporary script data (default: $BASH4LLM_DIR/tmp).

    BASH4LLM_RUN_DIR
        Directory hosting runtime process files (default: $BASH4LLM_DIR/var/run).

    BASH4LLM_LOCKS_DIR
        Isolated directory storing lock files (default: $BASH4LLM_RUN_DIR/locks).

    BASH4LLM_RATES_DIR
        Isolated directory tracking rate-limiting transaction timestamps 
        (default: $BASH4LLM_TMPDIR/rates).

    BASH4LLM_EXTRAS_DIR
        Active directory for accessing user extensions (default: $BASH4LLM_DIR/extras).

    PROVIDERS_DIR
        Directory containing external provider modules (default: $BASH4LLM_EXTRAS_DIR/providers).

    THREAD_CACHE_DIR
        Directory hosting temporary thread message caches (default: $BASH4LLM_CONFIG_DIR/thread_cache).

  Section 3: Resource Configuration and Synchronization
    MODELS_FILE
        Path of local file storing retrieved models (default: $BASH4LLM_MODELS_DIR/<provider>.txt).

    MAX_MODELS
        Numeric limit of models manageable locally (default: 200).

    PROVIDER_FILE
        Path of file persisting the active provider name.

    MODELS_LOCK
        Exclusive lock regulating updates to models files (default: $BASH4LLM_LOCKS_DIR/models.lock).

    HISTORY_LOCK
        Exclusive lock preventing corruption during history and thread operations 
        (default: $BASH4LLM_HISTORY_DIR/history.lock).

    TMP_LOCK
        Exclusive lock regulating temporary directory allocation (default: $BASH4LLM_LOCKS_DIR/tmp.lock).

    BASH4LLM_LOCK_TIMEOUT_TMP
        Timeout in seconds to acquire temporary directory locks (default: 10).

    BASH4LLM_LOCK_TIMEOUT_MODELS
        Timeout in seconds to acquire models file locks (default: 10).

    BASH4LLM_LOCK_TIMEOUT_HISTORY
        Timeout in seconds to acquire history and thread locks (default: 10).

  Section 4: History Maintenance (Rotation)
    BASH4LLM_ROTATE_HISTORY
        Boolean flag (1 or 0) enabling periodic history log rotation (default: 0).

    BASH4LLM_HISTORY_MAX_FILES
        Maximum threshold of files in history directory (default: 100).

    BASH4LLM_HISTORY_MAX_BYTES
        Cumulative byte limit for history files (default: 104857600, 100MB).

    BASH4LLM_HISTORY_KEEP_DAYS
        Retention limit in days for history files (default: 90 days).

  Section 5: Parameters, Rate Limiting and Security Policy
    BASH4LLM_RATE_LIMIT
        Numeric limit of allowed API requests per thread within a sliding 30-second 
        window (default: unlimited).

    BASH4LLM_AUTH_TOKEN
        Secret authorization token that bypasses local rate-limiting checks.

    FALLBACK_PAYLOAD
        Base64-encoded fallback payload generated by post-execution hooks on API failures.

    BASH4LLM_IGNORE_SEC_CHECKS
        Set to 1 to bypass strict POSIX file ownership and permission checks 
        (useful for WSL, Cygwin, or non-standard filesystems).

    ALLOW_API_CALLS
        External transaction gate flag. If set to 0, blocks network calls.

    BASH4LLM_SKIP_NETWORK
        Inhibits network calls and forces dry-run simulated execution.

    BASH4LLM_ENFORCE_NO_NETWORK_IF_QUIET
        If set to 1, blocks external connections whenever QUIET=1 is active.

    MAX_STAGE_BYTES
        Maximum byte limit for Base64 staging (default: 10485760, 10MB).

    DEBUG
        Diagnostic flag. Set to 1 to enable verbose debugging to stderr.

    BASH4LLM_LOG
        File path for structured chronological logging.

    BASH4LLM_SOURCE_ONLY
        Development guard. Set to 1 to exit immediately after bootstrap for testing.

EXIT CODES
    The runtime adopts a set of fixed numeric exit codes:

    10 (BASH4LLM_ERR_NO_API_KEY)
        Required API key is missing from environment and Vault in non-interactive session.

    11 (BASH4LLM_ERR_BAD_MODEL)
        Selected model is invalid, unsupported, or non-textual multimodal.

    12 (BASH4LLM_ERR_CURL_FAILED)
        Network request via curl failed due to connection issues or timeouts.

    13 (BASH4LLM_ERR_INVALID_JSON)
        JSON configuration or API response payload is corrupt or syntactically invalid.

    14 (BASH4LLM_ERR_NO_PROMPT)
        No prompt text or JSON payload was provided.

    15 (BASH4LLM_ERR_TMP)
        Unable to allocate secure temporary directory or lock acquisition failure.

    16 (BASH4LLM_ERR_API)
        Provider API rejected the request, returned HTTP error status, or yielded empty completion.

    17 (BASH4LLM_ERR_SEC)
        Security policy violation: insecure group/world writable file permissions, 
        symbolic link detected on critical path, or cryptographic SHA-256 module 
        integrity check failure against manifest.sha256.

OPERATIONAL MODES
    bash4llm implements five main operational modes:

  1. Single Request (Default)
    Acquires prompt, resolves provider and model, validates API keys, enforces rate 
    limiting and pre-execution hooks, executes query, and outputs response.

  2. Token Streaming (--stream)
    Establishes unbuffered connection, parses SSE line-by-line, outputs text 
    deltas immediately, and saves aggregated JSON response.

  3. Batch Processing (--batch)
    Scans prompt file line-by-line, executing isolated API calls per prompt.

  4. Interactive Chat (--chat, --tui)
    Launches full-screen interactive TUI REPL for multi-turn conversations.

  5. Interactive Provider Selection (--provider)
    Opens interactive selection menu to choose and persist active provider.

INTERNAL IMPLEMENTATION DETAILS
    This section describes core algorithms and security logics.

  1. Cryptographic Module Integrity and Manifest Verification
    External providers, hooks, and extensions undergo mandatory two-tier verification 
    via verify_module_integrity before loading: path security validation (ownership, 
    no group/world write) and SHA-256 hash matching against extras/manifest.sha256. 
    Mismatches abort execution with exit code 17.

  2. Thread PII Anonymization and Locking
    Raw thread IDs are cryptographically hashed using SHA-256 via anonymize_thread_id 
    producing $SAFE_THREAD_ID. All disk files (.ndjson, .json, locks) use this 
    anonymized hash, preventing sensitive PII leakage to the file system.

  3. Local Sliding Window Rate Limiting
    check_local_rate_limit tracks request timestamp files under $BASH4LLM_RATES_DIR/$SAFE_THREAD_ID. 
    Requests exceeding $BASH4LLM_RATE_LIMIT within a 30-second window are blocked with exit code 17.

  4. Cryptographic Key Vault Architecture
    When OpenSSL is active, provider API keys are encrypted using Master Key wrapped 
    AES-256-CBC with PBKDF2 (100,000 iterations) and salt. Session context token 
    $_B4L_RT_CTX holds unlocked vault keys in RAM to bypass password prompts.

  5. Prompt Assembly Priority Chain
    Assembly of $CONTENT: Structured JSON ($JSON_INPUT) -> files via -f -> 
    templates ($TEMPLATE) -> standard input ($STDIN_CONTENT) -> positional arguments.

TROUBLESHOOTING AND DIAGNOSTICS
    Anomalous Empty Responses (Empty Completion)
        Detected automatically via detect_empty_edge_case when HTTP 200 is 
        returned with zero tokens generated. Logs diagnostics and exits with code 16.

    Connection Failures or Lock Timeouts
        If exit code 15 occurs with timeout messages, verify no background 
        instances are holding locks, or increase BASH4LLM_LOCK_TIMEOUT_... values.
