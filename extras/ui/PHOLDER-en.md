### Essential Operational Rules (English, concise)

- **Mandatory escaping**: all textual values must go through `sanitize_param` and `html_escape` before insertion.  
- **Pre-generated HTML** (`MODEL_OPTIONS`, `PROVIDER_OPTIONS`, `CONV_LIST`, `CURRENT_CONV`, `LANG_OPTIONS`, `MODEL_LIST_SCROLL`, `MODEL_SELECT_OPTIONS`): **must NOT be double-escaped**; insert as trusted HTML generated by the server.  
- **Important distinction**:  
  - `MODEL_OPTIONS` = simple global model selector  
  - `MODEL_LIST_SCROLL` + `MODEL_SELECT_OPTIONS` = produced by `build_model_list_and_select()` for the Settings page  
- **API key**: `{{API_KEY_FIELD}}` is only for the password input; never logged or exported in templates.  
- **Positional placeholders**: `{{1}}..{{N}}` are escaped automatically by `render_template`.  
- **Localization**: every `{{TXT_*}}` is resolved from `gui-lang.conf` for the current language.  
- **Validation**: `LANG_CODE` and `THEME` must be validated before writing to config files.  
- **Boolean strings**: `CONFIGURED` and `MODEL_WHITELIST_PRESENT` are literal `"true"` / `"false"` strings.
